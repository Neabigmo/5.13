# Submission pack overview

This folder is a working submission scaffold for a Machine Learning/Springer
submission. It contains:

- `codex_docs/`: detailed instructions for Codex to revise the paper, code, experiments, references, and figures.
- `manuscript/`: Springer Nature-style LaTeX manuscript using `sn-jnl`
  conventions, now synced to the active source in `paper/sections/`.
- `supplementary/`: reproducibility and environment scaffold.
- `editorial/`: cover letter, checklist, and availability statement templates.

Current packaging status:

- Active manuscript sections and bibliography have been ported into `manuscript/`.
- Generated figures/tables and the full-run manifest from
  `outputs/runs/ml_submission/finalrun_manifest/` are included.
- The one-command runner lives at `experiments/run_all.py` with configs under
  `experiments/configs/`.
- Data figures were regenerated by `experiments/nature_figures.py` using the
  local `external/nature-skills/skills/nature-figure/` conventions. Each figure
  has editable `.svg`, `.pdf`, and 300 dpi `.png` outputs.
- The LaTeX build path used for verification is
  `D:\texlive\2025\bin\windows`.

Springer template note: the official Springer Nature LaTeX author support page
currently points to the journal article template package marked December 2024.
The bundled `sn-jnl.cls` and `.bst` files in `manuscript/` were taken from that
package so the submission manuscript compiles locally.
