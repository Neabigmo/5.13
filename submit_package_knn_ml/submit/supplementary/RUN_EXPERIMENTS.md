# How to run experiments after revision

The repository now includes a config-driven runner:

```bash
# from repository root
python -m pytest -q
python experiments/run_all.py --config experiments/configs/ml_submission.toml
```

Notes:

- `ml_submission_smoke.toml` is the fast end-to-end validation config.
- `ml_submission.toml` is the full benchmark config and may take substantially longer.
- Each run writes to `outputs/runs/ml_submission/<timestamp>/`.
- The current submission package was synced from
  `outputs/runs/ml_submission/finalrun_manifest/`.
- Figures are regenerated by `experiments/nature_figures.py` using
  `external/nature-skills/skills/nature-figure/` conventions and are emitted as
  editable `.svg`, `.pdf`, and 300 dpi `.png`.

Run layout:

```text
outputs/runs/ml_submission/<timestamp>/
  raw/
    witnesses/
    experiments/
  rendered/
    figures/
    tables/
  summaries/
  manifests/
```

The synced submission scaffold currently includes the full-run manifest at:

```text
submit/logs/run_manifest.json
```

Compile commands used for the current package:

```powershell
& 'D:\texlive\2025\bin\windows\latexmk.exe' -pdf -interaction=nonstopmode -halt-on-error paper\main.tex
& 'D:\texlive\2025\bin\windows\latexmk.exe' -cd -pdf -interaction=nonstopmode -halt-on-error submit_package_knn_ml\submit\manuscript\main.tex
```
